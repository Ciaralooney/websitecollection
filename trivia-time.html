Trivia Time

Trivia Time is a true or false trivia game for the Yoto Player. When the card is inserted 10 random questions are picked and they are read out one at a time. This is an audio game that uses the Yoto's two physical buttons. A python server streams the audio live and keeps track of the players score.

Genre: Trivia game

Platform: Yoto Player

Team: Solo

My roles: Game Designer, Programmer

Status: Released

Read the design report View the code

Working with the two buttons

Result: A device built for skipping tracks plays a real trivia game. The left button is false and the right is true. When they're press the game responds immediately to whichever you press and keeps track of the score in the background.

Problem: The Yoto's buttons only move forward and backward through the tracks on a card. There's no way to setup the tracks on a card to have multiple options. This means it wasn't possible to ask a question and jump to two different tracks depending on the answer the player gives.

Solution: Instead of relying on the card itself or the yoto website I setup a Python server that listens to live MQTT events from the player. It works out which button was pressed based on where the player lands, it updates the score based on what the player chose and then immediately overrides playback to send them to the right track.

Live audio

Result: Your score is read out with feedback as the game progresses. The ten random questions each game means no two playthroughs sound the same.

Problem: Pre-recorded tracks couldn't dynamically read out the players score and give them feedback based on how they're doing.

Solution: Questions and feedback are streamed in real time using a text to speech voice. The audio is generated live as you play. Ten questions per game felt like the right length, long enough to feel like a proper game but short enough that an 8 to 12 year old won't lose interest halfway through.

What I learned

The hardest part of this project was figuring out how to get real interactivity out of hardware that was never built for it. The Yoto's buttons only move forward and backward through the tracks on a card. They don't support jumping to two different places from one point. So after some research I found I could make a server that listens to live MQTT events from the player. It works out which button the player pressed based on where they land. It then immediately overrides playback to send them to the next question while keeping track of their score. Moving all the game logic to the server is what made the project possible at all.
